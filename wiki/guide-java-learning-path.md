# Java 학습 경로 — 3단계 로드맵

> [[src-java-study-2024-2025]]의 원본 교재를 **"언어 → 프레임워크 → 고급"** 세 단계로 재구성했다. 챕터 본문은 위에서 아래로 읽는 레퍼런스이고, 트랙(학습 코스)은 같은 내용을 미니프로젝트로 손에 익히는 동선이다.

**학습 원칙**: 문법을 외우는 게 아니라 **왜 그렇게 동작하는지 설명**할 수 있는 상태를 목표로 한다. 각 단계 끝의 🛠 미니프로젝트로 굳힌다.

---

## 🗺️ 3단계 한눈에

세 단계는 순수 언어에서 시작해, 그 위에 프레임워크를 얹고, 마지막에 품질과 성능으로 마감하는 선형 흐름이다.

```
1단계 Java Core ──> 2단계 Spring & 웹 ──> 3단계 고급·품질
 (언어 기초 체력)     (웹 애플리케이션)       (테스트·성능)
```

순수 Java로 객체지향과 입출력까지 다진 다음(1단계), Spring으로 웹을 만들고 DB를 연동하며(2단계), 테스트와 JVM 튜닝으로 견고하게 마무리한다(3단계). 앞 단계가 뒷 단계의 전제이므로 건너뛰지 말고 순서대로 따라간다.

---

## 🧭 본문 학습 순서

챕터 본문은 번호순(ch00 → ch01 → … → ch11)으로 "다음 장" 링크가 이어지며, 그 번호가 곧 학습 순서다:

```
환경 → 문법·객체 → 컬렉션·함수형 → 객체지향 설계 → 입출력·네트워크
   → Spring → 데이터·SQL → 서버·인증
   → 테스트 → JVM·성능 → 부록
```

[[java-study-ch00]]의 안내부터 시작해 위 순서대로 읽으면 1단계 → 2단계 → 3단계가 자연스럽게 이어진다.

---

## 📚 학습 코스(트랙)와의 관계

3단계가 **지식 계층**(무엇에 속하나)이라면, T1~T5 트랙은 **미니프로젝트 중심 학습 동선**(어떻게 손에 익히나)이다. 처음 배우는 사람은 트랙을 따라가고, 필요한 주제를 레퍼런스로 찾을 땐 3단계 본문을 쓰면 된다.

| 학습 코스(트랙) | 해당 단계 |
|------|----------|
| [[guide-java-track1-basics]] — Java 기초 다지기 | 1단계 |
| [[guide-java-track2-design]] — 객체지향 설계로 응용 | 1단계(설계) + 3단계(JVM 성능) |
| [[guide-java-track3-io-network]] — 입출력과 네트워크 | 1단계 |
| [[guide-java-track4-spring-web]] — Spring 웹 애플리케이션 | 2단계 + 3단계(테스트) |
| [[guide-java-track5-deep-dive]] — 심화·워크북·종합 | 3단계 |

> 트랙 T2·T4가 두 단계에 걸치는 이유는, 미니프로젝트 동선상 설계 트랙에서 JVM을, Spring 트랙에서 테스트를 함께 다루기 때문이다. 지식 계층(3단계)에서는 JVM·테스트가 고급(3단계)에 속한다.

## 💡 이 경로의 특징

- **이론·방법론 연결**: 각 트랙에 💡 팁 박스로 위키의 5권 도서 강의([[entity-object]]·[[entity-effective-java]]·[[entity-refactoring]]·[[entity-clean-code]]·[[entity-tdd]])와 개념 페이지를 연결하고, "자세히 →" 링크로 바로 깊이 들어갈 수 있게 했다.
