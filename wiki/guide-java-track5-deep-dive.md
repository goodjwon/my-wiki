## 💡 이론·방법론 연결 (팁)

> 💡 **면접 — "설명할 수 있어야 안다"**
> 11.13·11.14는 암기가 아니라, T1~T4 개념을 *한 문장으로 말하는* 연습이다. 위키의 다른 개념 페이지와 짝지어 보라.
> → 자세히: [[concept-oop]] · [[concept-spring-core]] · [[entity-jvm]]

> 💡 **협업 — Git은 안전망**
> 11.15·11.18을 익히면 미니프로젝트를 브랜치+PR로 안전하게 진행한다. 하네스의 "거부 신호"와 같은 맥락.
> → 자세히: [[guide-harness-module3]] (guard.sh로 main 직접 push 차단)

> 💡 **품질 — 워크북은 리팩터링 연습장**
> 11.90 JVM 워크북·실전문제는 "동작하는 코드 → 더 나은 코드"를 반복하는 장. 악취를 찾아 고치는 눈을 기른다.
> → 자세히: [[entity-refactoring]] · [[guide-code-authoring-and-review]]

## 🛠 미니프로젝트 — 종합 토이프로젝트

T1~T4를 모두 동원하는 가장 큰 산출물. 하나를 골라 끝까지:

1. **블로그 작성하기 (11.25 / 11.23)** — Java/JSP(Model 1) 기반 CRUD 블로그. 웹의 전 과정을 한 번에.
2. **온라인 게임 토이 프로젝트 (11.19)** — 객체 협력·파일/네트워크·동시성 종합.
3. **메모 앱 (11.26)** — T2에서 만든 걸 확장: 저장·검색·테스트 가능 구조로.

**따라하는 법**:
```text
1) 요구사항을 기능 단위로 쪼개 task-list 작성
2) Git 브랜치 + 작은 커밋 (11.18)
3) 각 기능: 실패 테스트 → 구현 → 리팩터 (TDD)
4) T4 패턴 적용: 트랜잭션·예외·계층 분리
5) README에 "무엇을 왜 어떻게" 기록 → 포트폴리오(11.17)
```
실습 본문·시나리오 → [[java-study-ch11]](11.25·11.23·11.19·11.26).
