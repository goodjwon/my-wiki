# 실습 2-2: Before / After 비교 프롬프트

## 목적
CLAUDE.md 적용 전후로 동일한 태스크를 실행해서 실제 개선을 수치로 확인합니다.

---

## 실행 순서

### Step 1: CLAUDE.md 적용 확인

```
현재 프로젝트 루트에 CLAUDE.md가 있는지 확인하고,
첫 번째 섹션을 읽어서 내용을 요약해줘.
```

---

### Step 2: Module 1에서 했던 태스크 A 재실행

```
[CLAUDE.md 적용 상태]

다음 요구사항을 구현해줘:

계약(Contract) 엔티티에 '계약 담당자 이메일(managerEmail)' 필드를 추가해야 해.
이메일 형식 검증이 필요하고, null 허용 안 됨.
관련 API도 업데이트해줘.

시작 전 CLAUDE.md의 작업 전 체크리스트를 먼저 확인하고,
단계별 계획을 먼저 제시해줘.
```

---

### Step 3: diff 비교 프롬프트

```
방금 완료한 구현의 git diff를 확인해줘.

다음 항목을 비교해서 표로 정리해줘:

| 항목 | Before (Module 1) | After (Module 2) |
|------|-------------------|------------------|
| DTO 없이 Entity 노출 | Y/N | Y/N |
| Migration 파일 생성 | Y/N | Y/N |
| 테스트 작성 | Y/N | Y/N |
| 불필요한 코드 추가 줄 수 | N줄 | N줄 |
| 메시지 교환 횟수 | N회 | N회 |
| 레이어 의존 방향 준수 | Y/N | Y/N |

그리고 CLAUDE.md 중 이번 태스크에서 실제로 작동한 규칙을 나열해줘.
작동하지 않았거나 보완이 필요한 규칙도 말해줘.
```

---

### Step 4: CLAUDE.md 보완

```
비교 결과를 바탕으로 CLAUDE.md 섹션 11(누적 실패 패턴)을 업데이트해줘.

이번 실습에서 발견된 새 패턴이 있으면 추가하고,
불필요하거나 이미 자동으로 지켜지는 규칙은 제거를 제안해줘.
```
